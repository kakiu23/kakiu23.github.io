Codex 配置第三方中转站 API 教程

这篇文章记录一下在 macOS 上通过 Codex++ 管理工具 给 Codex 配置第三方中转站 API 的过程。

Codex++ 是 Codex App 的外部增强启动器和管理工具。它不修改 Codex App 原始安装文件，而是通过外部 launcher 启动 Codex，并使用 Chromium DevTools Protocol 注入增强脚本。这里主要用到它的 中转注入 功能，把 Codex 的模型请求切到第三方兼容 API。

本文使用的是 Mac 电脑演示，Windows 用户整体流程类似，只是安装包和部分系统提示不一样。

1. 准备工作

开始之前，先准备好下面几样东西：

  1. 已经安装好的 Codex App。
  2. 一个可以正常登录的 ChatGPT / Codex 官方账号。
  3. 一个第三方中转站账号。
  4. 第三方中转站提供的 API Key。
  5. 第三方中转站提供的 Base URL。
  6. Codex++ 管理工具安装包。

第三方中转站只要兼容 Codex 所需的 OpenAI API 协议即可，不一定固定为某一家。实际配置时需要重点确认两个参数：

  • Base URL：中转站的接口地址，例如 https://example.com/v1
  • API Key：中转站后台生成的密钥，一般以 sk- 开头

2. 下载 Codex++

打开 Codex++ 的 GitHub Releases 页面，下载适合自己电脑的安装包：

  • Intel 芯片 Mac：下载 macos-x64.dmg
  • Apple Silicon 芯片 Mac：下载 macos-arm64.dmg

如果你不知道自己的 Mac 是 Intel 还是 Apple Silicon，可以点击左上角苹果图标，进入“关于本机”查看芯片信息。

3. 安装 Codex++

下载完成后，双击打开 .dmg 安装包，把里面的应用拖到 Applications 应用程序目录。

安装完成后，应用程序里会出现两个入口：

  • Codex++：静默启动入口，不显示管理界面，只负责启动 Codex 并注入增强功能。
  • Codex++ 管理工具：控制面板，用来配置中转注入、检查状态、查看日志、管理增强功能等。

4. 先确认 Codex 官方登录状态

中转注入模式的逻辑是：

  • 官方 ChatGPT / Codex 登录态继续负责账号能力和插件入口。
  • 第三方中转站只接管模型请求使用的 Base URL、API Key 和模型名称。

所以在配置中转站之前，建议先正常打开 Codex App，并确认自己已经登录官方账号。

检查点：

  1. Codex 能正常打开。
  2. 账号状态正常。
  3. 插件入口或账号相关功能可用。

5. 获取第三方中转站 API 信息

登录第三方中转站后台，找到 API Key 管理页面，新建或复制一个 API Key。

6. 打开 Codex++ 管理工具

打开 Codex++ 管理工具，进入中转注入相关页面。

这里建议先看一下管理工具里的诊断状态，确认 Codex++ 能正常识别本地 Codex 环境。

7. 添加中转配置

在“供应商配置”页面里添加一个新的供应商配置，然后填写：

  • 配置名称：自己能看懂即可，例如 我的中转站
  • Base URL：中转站提供的接口地址
  • API Key：中转站生成的密钥
  • 配置模型：中转站支持的模型名
  • 接入模式：纯API

如果页面里有协议或 Wire API 选项，优先选择 responses 或平台文档推荐的 Codex 兼容协议。

配置填写完成后，选择刚刚添加的中转配置，点击应用中转注入。

9. 通过 Codex++ 启动 Codex

配置完成后，不要直接从原版 Codex 图标启动，而是从 Codex++ 入口启动。

原因是 Codex++ 需要在启动 Codex 的时候注入增强脚本和中转配置。如果直接打开原版 Codex，可能看不到 Codex++ 菜单，相关功能也不会生效。

启动后可以检查顶部菜单栏或设置面板里是否出现 Codex++ 相关入口。

11. 如何切回官方模式

如果后面想恢复官方 ChatGPT / Codex 登录模式，可以重新打开 Codex++ 管理工具，进入“供应商配置”页面，使用默认中转配置就是原来的登录模式。

12. 常见问题

12.1. Codex++ 菜单没有出现

优先检查是不是从 Codex++ 入口启动的 Codex，而不是从原版 Codex 图标启动。

如果确认入口没问题，可以打开管理工具里的“诊断”或“日志”页面，看注入状态是否正常。

12.2. 提示 API Key 无效

一般检查这几项：

  1. API Key 是否复制完整。
  2. API Key 前后有没有多余空格。
  3. 中转站账号余额是否充足。
  4. 当前模型是否在中转站支持列表里。
  5. Base URL 是否填错。

12.3. 请求失败或一直转圈

可以按下面顺序排查：

  1. 先确认网络能访问第三方中转站。
  2. 确认 Base URL 是否包含正确的 /v1。
  3. 换一个更常见的模型名测试。
  4. 打开 Codex++ 管理工具查看日志。
  5. 清除 API 模式后重新配置一次。

12.4. 插件市场还是提示需要登录

中转注入模式下，官方登录态仍然很重要。先回到原版 Codex 或官方登录页面，确认 ChatGPT / Codex 账号已经登录成功，再回到 Codex++ 管理工具应用中转注入。

12.5. 可以配置多个中转站吗

可以。Codex++ 支持多个中转配置，可以在管理工具里添加多个配置，然后选择当前要使用的配置进行应用。

13. 总结

整个流程可以概括为：

  1. 安装 Codex++。
  2. 确认 Codex 官方账号已经登录。
  3. 在第三方中转站获取 Base URL 和 API Key。
  4. 在 Codex++ 管理工具里添加中转配置。
  5. 应用中转注入。
  6. 通过 Codex++ 入口启动 Codex。
  7. 发消息测试是否走第三方 API。

配置成功后，Codex 的账号能力仍然由官方登录态负责，模型请求则走第三方中转站。这样既保留 Codex 原有体验，也可以灵活使用不同中转站提供的模型和价格方案。
