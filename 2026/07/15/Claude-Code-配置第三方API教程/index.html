Claude Code 配置第三方 API 教程

这篇文章记录一下如何使用 CC Switch 给 Claude Code CLI 配置第三方 API。

CC Switch 是一个跨平台的 AI 编程 CLI 统一管理工具，支持 Claude Code、Codex、Gemini CLI、OpenCode、OpenClaw 和 Hermes Agent。它的核心思路是把不同工具的 provider 配置统一管理起来，切换时不需要手动改一堆 JSON、TOML 或 .env 文件。

官方文档链接：https://ccswitch.io/zh/docs?section=getting-started

1. 先准备好这些东西

开始之前，先准备好下面几项：

  1. 已经安装好的 Claude Code CLI。
  2. 一个可用的第三方 API 账号。
  3. 第三方服务商提供的 API Key。
  4. 第三方服务商提供的 Base URL 或 Endpoint。
  5. 安装好的 CC Switch。

一般来说，第三方 API 至少会提供这三个关键信息：

  • Base URL：接口地址，例如 https://example.com/v1
  • API Key：访问密钥
  • Model：你要在 Claude Code 里使用的模型名

如果服务商已经提供了适配 Claude Code 的预设配置，直接用预设会最省事。如果没有，也可以手动创建自定义 provider。

2. 安装 CC Switch

CC Switch 官方支持 Windows、macOS 和 Linux。

macOS 推荐直接用 Homebrew：

如果你习惯手动安装，也可以去官方 Releases 页面下载 dmg 安装包。

Windows 和 Linux 用户也可以直接从 Releases 页面下载对应安装包，官方文档里已经把各平台安装方式列得比较清楚。

3. 新增第三方 Provider

打开 CC Switch 后，点击右上角的加号按钮添加一个 provider。

这里有两种方式：

  1. 直接选择第三方服务商预设。
  2. 新建一个自定义配置。

如果服务商在预设列表里，优先选预设。这样可以少填很多字段，也更不容易出错。

如果没有预设，就手动填下面这些内容：

  • Provider 名称：自己能看懂就行，例如 我的第三方 Claude
  • Base URL：服务商提供的接口地址
  • API Key：服务商生成的密钥
  • Model：要用的模型名

4. 启用这个 Provider

配置填好后，选中刚才创建的 provider，然后点击 Enable。

在系统托盘菜单里也可以直接点 provider 名称切换，官方文档里提到这是即时生效的方式。

对于 Claude Code，CC Switch 官方说明里写得很明确：切换 provider 不需要重启终端或 CLI，这点比很多其他工具方便。

启用成功后，直接打开 Claude Code CLI，发一条简单消息测试：

如果能正常回复，说明第三方 API 已经接通了。

6. 如何切回官方登录

如果后面想恢复 Claude 官方登录模式，也很简单：

  1. 在 CC Switch 里再添加一个 Official Login 预设。
  2. 切换到这个预设。
  3. 重启 Claude Code CLI。
  4. 按 Claude Code 自己的登录 / OAuth 流程重新登录即可。

官方文档里提到，切回官方模式后，可以在官方 provider 和第三方 provider 之间自由切换。

7. 常见问题

7.1. 切换后没生效

先检查这几项：

  1. 是否真的点了 Enable。
  2. 选中的 provider 是否就是刚才配置的那个。
  3. Base URL 是否写对，尤其是有没有漏掉 /v1。
  4. API Key 前后有没有多余空格。
  5. 第三方服务商账号是否还有余额。

7.2. Claude Code 需要重启吗

官方文档写的是：Claude Code 支持 provider 数据的热切换，一般不需要重启。

如果你发现没变更，优先重新打开一次 Claude Code，再看是否已经切到新 provider。

7.3. 可以同时管理多个第三方 API 吗

可以。

CC Switch 本身就是 provider 管理器，你可以为不同服务商分别建配置，然后按需切换。官方还提供了 Shared Config Snippet 之类的能力，方便把通用参数在不同 provider 之间复用。

8. 总结

整个流程其实很简单：

  1. 安装 CC Switch。
  2. 准备好第三方 API 的 Base URL、API Key 和模型名。
  3. 在 CC Switch 里新增 provider。
  4. 启用这个 provider。
  5. 直接打开 Claude Code 测试。

如果你只是想让 Claude Code CLI 快速切到第三方 API，CC Switch 的优势就是配置集中、切换快，而且 Claude Code 还支持热切换，不用反复重启终端。
